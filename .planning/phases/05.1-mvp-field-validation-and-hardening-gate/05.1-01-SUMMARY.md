## Accomplishments

- **Task 1 — SessionReplayDecoder:** A pure `commonMain` object (zero Compose/platform/Okio imports) whose `decodeForReplay(exportedBytes, config)` decodes a `TimingSessionPayloadV1` with the SAME `FileSessionStore.canonicalJson` `JsonExportService` encodes with, maps samples via the existing `LocationSampleDto.toModel()`, rebuilds the course via the direction-aware `courseFromTrack`, and runs `ReplayRunner`. Returns a sealed `ReplayDecodeResult.Decoded/Corrupt/NoCourse`; serialization failures are caught and converted to `Corrupt`, never re-thrown (T-5.1-01). Tests prove decoded output is structurally identical to a direct `ReplayRunner` run (lapCount + sectorEvents + finalState), garbage and truncated bytes yield `Corrupt`, and a degenerate start/finish yields `NoCourse`.
- **Task 2 — GPS fixtures:** `degradationDuringTiming()` (clean oval whose accuracy collapses to 48 m across one contiguous mid-lap span, position kept on-track, D-24) and `closedCoursePerpendicular()` + `closedCoursePerpendicularStartFinish()` (five ~12-pts/loop oval crossing a perpendicular north-south start/finish between samples, forcing interpolation, D-19/D-24). Both registered in `scenario(id)`; tests prove determinism across repeated calls, the contiguous >25 m degraded span, and a 4-lap reconstruction through `ReplayRunner`.

## Decisions Made

- **Host-test task name corrected:** `:shared:testAndroidHostTest`, not `:shared:testDebugUnitTest`. Recorded so every downstream Phase 5.1 plan uses a working command.
- **NoCourse made reachable:** Because `TimingSession.startFinish` is non-nullable, the `courseFromTrack`-returns-null path can never fire from a structurally-valid decoded payload. The decoder additionally treats a degenerate (zero-length, `pointA == pointB`) start/finish as `NoCourse`. This realizes the plan's NoCourse contract honestly and hardens the untrusted-bytes boundary (a zero-length line cannot define a crossing, so a silent 0-lap replay would be misleading).
- **Perpendicular replay uses learned direction:** The 3-arg `courseFromTrack(..., Recorded)` stamps an explicit accepted crossing sign that is enforced even under lenient config, which rejected the fixture's crossings. The 2-arg overload (no explicit sign, direction learned from the first crossing) is used so the test exercises the intended low-frequency interpolation regime; direction enforcement is covered by the existing `CourseDirectionReplayTest`.
