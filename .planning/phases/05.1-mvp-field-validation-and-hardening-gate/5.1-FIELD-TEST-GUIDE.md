# 5.1 Field-Test Guide — Run Sheet & Rubric

**Use this outdoors. Brief on purpose.** Full reasoning + decision refs: `5.1-UAT.md`. Record results in: `5.1-FIELD-TEST-LOG.md`.

**Goal:** 5 valid closed-course sessions (3 on Primary phone, 2 on Secondary) where the app got every lap count right. That decides Go / No-Go.

---

## ⚡ Quick card (memorize these 3 numbers)

App must show **Ready before you start timing.** Ready = all of:

| Accuracy | Fix age | Sample rate | + course / start-finish / direction set |
|---|---|---|---|
| **≤ 25 m** | **≤ 15 s** | **≥ 1.0 Hz** | all selected & valid |

**If not Ready → do not time.** (You may do *raw recording* for diagnosis, but it does NOT count as a session.)

A session counts only if **all 4** are saved: ① JSON+GPX export ② replay result ③ field notes ④ start/finish video.

**Lap count must be 100% correct.** One wrong count on a valid session = No-Go for that run.

---

## A. Before you leave (at the desk)

- [ ] `./gradlew :shared:testAndroidHostTest` → green
- [ ] `./gradlew :androidApp:assembleDebug` → success
- [ ] Install APK on **both** phones: `adb -s <serial> install -r androidApp/build/outputs/apk/debug/androidApp-debug.apk`
- [ ] Pack: 2 Android phones (Primary + Secondary), mount, **second camera** for start/finish video, chargers/power bank.
- [ ] Note each phone's name/model + OS version (write "Primary" / "Secondary").

## B. At the course — per phone (≈5 min each)

Run once on each phone before timing:

- [ ] **S1** App opens to the dash, no crash.
- [ ] **S2** Grant precise location; a real GPS fix appears.
- [ ] **S3** Live speed, accuracy (m), and update rate are all visible and changing.
- [ ] **S4** Rotate portrait ↔ landscape; dash renders both, controls reachable.
- [ ] **S6** Settings: **Phone GPS ↔ Simulated** — dash label honestly tracks the source (no silent fallback). Leave it on **Phone GPS**.

## C. Mounted-display glance check (do once, ~2 min)

Mount the phone. From a **1-second glance**, can you read all 7? Check in **portrait + landscape**, and again in **daylight + low-light**:

`① current lap time  ② delta-to-best (or --)  ③ lap count  ④ last/best lap  ⑤ GPS Ready/poor  ⑥ raw-recording on/off  ⑦ timing on/off`

- [ ] All 7 legible ≤1s in all 4 cells (portrait/landscape × day/low-light).

---

## D. Run a session — repeat ×5 (3 Primary, 2 Secondary)

1. Set up the course/profile; confirm start/finish line + direction.
2. **Wait for the app to show Ready** (the 3 numbers above). Do not start until green.
3. Start a **second-camera video** aimed at the start/finish line.
4. Start timing in the app. Drive a known number of laps — **count them yourself out loud / on camera**.
5. Stop timing. Save the session.
6. **Capture the 4 evidence items** (Section E) before moving on.

> Wrong-course **"Still use this track" override** is allowed, but it bypasses Ready — note it in the log and make sure it's visible in Review/export.

## E. After each session — capture evidence (all 4 or it doesn't count)

- [ ] ① **Export** the session as **JSON** *and* **GPX**.
- [ ] ② **Replay**: run the JSON through `SessionReplayDecoder.decodeForReplay` → must return **Decoded**, identical across 2 runs, lap count matches your tally.
- [ ] ③ **Notes**: device + Ready-passed + lap count pass/fail (extras only if something went wrong).
- [ ] ④ **Video**: keep the start/finish clip.
- [ ] Then on-device smoke: **Review tab** shows the session (lap list, best lap, duration, GPS quality) + track trace renders; save a lap as **reference/best**; confirm JSON re-replays and GPX opens as a real track.

## F. If a lap/timing looks wrong — replay first, always

Never blame from memory. **Export → `decodeForReplay` → run twice.** Then assign the cause to exactly one of:
`data quality · line placement · interpolation · Ready classification · course selection · implementation logic`.
A replay that drifts or returns Corrupt/NoCourse *is* the finding. Add every export as a permanent replay fixture and link it in the log.

---

## G. Rubric — what the result means (decides the gate)

Score each session, then read the outcome.

**Per-session validity** (all must be true, else the session is disqualified — redo it):
- [ ] App was Ready before timing · [ ] JSON+GPX export saved · [ ] Replay Decoded + deterministic · [ ] Notes saved · [ ] Start/finish video saved

**Per-session pass:**

| Check | Standard | Weight |
|---|---|---|
| **Lap count vs your tally** | **100% correct** | **HARD — any miss = No-Go run** |
| Lap time vs *readable* video crossing | **≤ 0.5 s** | Target (not an auto-blocker); skip unreadable crossings |
| Review + JSON/GPX export smoke | all pass | Required for the session |

**Phase outcome:**

| Outcome | When |
|---|---|
| ✅ **Go** | 5 valid sessions, **every** lap count 100%, all replays deterministic, Review/export smoke passes. (Code + UI audits already clear.) |
| 🔧 **Hardening Required** | A fixable, diagnosable defect blocks Go (Ready/course-selection/replay drift) — but no lap-count failure. Fix, rerun affected sessions. |
| ⛔ **No-Go** | A valid session has a wrong lap count that can't be made trustworthy after replay/code diagnosis, or Ready can't prevent untrustworthy timing. |
| ⏳ **Pending** | Can't get a second phone or closed course yet. **Not a failure** — phase stays blocked until evidence exists (D-54). |
