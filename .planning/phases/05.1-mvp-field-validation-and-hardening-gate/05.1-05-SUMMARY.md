# Phase 5.1 Plan 05: UI Tokenization Hardening & Six-Pillar Re-Audit Summary

**A bounded tokenization hardening that routes every color, font size, and structural padding on the Drive dash and shell through `MaterialTheme` semantic tokens (`ui/Theme.kt` + `ui/Spacing.kt`) instead of 30+ inline hex and 100+ inline `.sp`/`.dp`, replaces the brittle `if (compact) 42.sp else 54.sp` glance sizing with autoSize-bounded display roles that never clip, and re-scores all six D-36 pillars against the UI-SPEC — lifting Visual Polish/Accessibility (1/4 -> 3/4) and Motion/Glance (2/4 -> 3/4) so every pillar is now >= 3/4 (overall 19/24). The dash information hierarchy and the manual orientation toggle are unchanged; `:androidApp:assembleDebug` and the full host suite are green.**

## Accomplishments

- **Task 1 — Token systems wired into MaterialTheme:** `ui/Theme.kt` defines
  `lapsightTypography` (UI-SPEC roles: Display 56/600/1.0, Heading 24/600/1.2,
  Label 14/600/1.2, Body 16/400/1.5, plus the intermediate steps the dash uses) and
  the expanded dark/light color schemes with `tertiary`/`error`/`errorContainer`
  state roles. `ui/Spacing.kt` defines the 4dp `Spacing` scale (xs 4 … xxxl 64) via
  `LocalSpacing`. `App.kt` passes `typography = lapsightTypography` and wraps the
  shell in `CompositionLocalProvider(LocalSpacing provides Spacing())`; the dark
  racing theme stays the default (D-26).
- **Task 2 — Dash + shell tokenized, glance sizing hardened:** All inline
  `Color(0xFF…)` and `fontSize`/`.sp` were removed from `DriveScreen.kt` and
  `AppShell.kt` (verified: 0 remaining). Colors route through `colorScheme` semantic
  roles; type through `typography` roles; structural padding/gaps through
  `LocalSpacing`. The hero readouts (current lap / speed / delta) now use
  `TextAutoSize.StepBased` bounded by `displayLarge`/`displaySmall`/`displayMedium`
  with `maxLines = 1`, so they shrink to fit and cannot clip on foldables / edge
  aspect ratios. The `isCompactLandscape` branch and the manual orientation toggle
  are untouched; no sensor rotation introduced (project memory honored).
- **Task 3 — Six-pillar re-audit:** `5.1-UI-REVIEW.md` rewritten with all six D-36
  pillars scored 1-4 against the UI-SPEC, with per-pillar evidence. Visual
  Polish/Accessibility 1/4 -> 3/4 and Motion/Glance 2/4 -> 3/4 are justified by the
  tokenization and autoSize work; Information Hierarchy lifted 3/4 -> 4/4 (now
  semantically encoded). Overall 19/24, every pillar >= 3/4 (D-37). On-glass
  daylight/low-light/motion glance confirmation is explicitly deferred to the Plan 07
  display UAT.

## Intentional Residual Literals (documented per plan output spec)

These are deliberately NOT tokenized because they are component dimensions, not
design-token color/type/spacing:

- **Component dimensions:** button heights (`52/54/48/44/40/36.dp`), icon sizes
  (`24/28.dp`), corner radii (`RoundedCornerShape(6/8.dp)`), stroke widths
  (`1.dp` borders, Canvas stroke widths), `heightIn(max = 420.dp)`, `SpeedTrace`
  height (`72.dp`).
- **Baseline nudges:** a couple of small `bottom = 2.dp` text-baseline offsets in
  telemetry cells/units.
- **`PaddingValues(0.dp)`** content-padding resets on icon buttons.

No remaining inline `Color(0xFF…)` or `fontSize`/`.sp` exist on the dash or shell.
