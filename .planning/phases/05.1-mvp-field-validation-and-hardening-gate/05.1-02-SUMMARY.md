# Phase 5.1 Plan 02: Full-pipeline replay determinism suite Summary

**A standing N-run determinism gate over the full `SessionController`+`TimingSessionRecorder` pipeline (completed laps, `completedSectorResults`, the ordered `LiveDeltaSnapshot` state-transition sequence, and the resolved `CourseCompatibilityKey`/direction — all byte-identical across runs and across export→decode→replay), plus widening of five pre-existing replay/recovery/ghost/direction tests beyond the legacy `finalState`-only check.**

- **Host-test task:** `:shared:testAndroidHostTest` (per Plan 01; `:shared:testDebugUnitTest` does not exist)

## Accomplishments

- **Task 1 — `FullPipelineDeterminismTest`:** New `commonTest` suite with a `runOnce(scenarioId, direction)` helper that builds a Track over the canonical oval (perpendicular start/finish + a one-loop oval reference line for the live matcher), runs the full `SessionController` pipeline over `GpsFixtureLibrary.scenario(id)`, and captures an immutable `PipelineCapture` (completed laps, `completedSectorResults`, the ordered `LiveDeltaSnapshot` sequence via the change-detection idiom, and `CourseCompatibilityKey`). One `@Test` per regime — clean, noise/drift, dropped-low-frequency, degradation-during-timing, closed-course-perpendicular, outlier-recovery, and a Recorded-vs-Reverse direction-decision case — asserts `runOnce == runOnce` by data-class structural equality. A dedicated test exports a built session via `JsonExportService`, decodes it twice through `SessionReplayDecoder.decodeForReplay`, and asserts the two replays are identical (D-25/D-28). Lap times are compared as exact `Long` millis; no tolerances anywhere (D-27).
- **Task 2 — widened existing tests:**
  - `ReplayTest.replayIsDeterministic` now asserts the full ordered per-sample state sequence, the completed-lap list, and the ordered sector-event sequence (was `finalState` only — the exact gap this plan closes).
  - `CompleteSectorReplayTest` gains an N≥3-run assertion that `completedSectorResults` is byte-identical across independent runs.
  - `LiveDeltaEngineTest` gains an ordered availability/state-transition-sequence determinism test (Available + poor-GPS suppression + recovery), captured with the change-detection idiom.
  - `DraftRecoveryTest` gains an equivalence test: a resumed mid-draft session reaches the identical engine state as a fresh replay of the same samples (driven by `ReplayFixtures.multiLapLoop` so real laps are produced on the DEMO course).
  - `CourseDirectionReplayTest` promotes the Reverse-lap and turnaround cases to two-run determinism assertions (lap count + `completedSectorResults` + full state).

## Decisions Made

- **Reverse direction drives the oval fixtures.** `LapEngine.explicitDirection = (course.acceptedStartFinishSign != null)`, and the direction gate fires on `(explicitDirection || config.enforceDirection)` — so a 3-arg (Recorded/Reverse) course enforces its accepted side even under `lenientForTests()`. A probe confirmed the oval's natural crossing is accepted by Reverse (clean=9, noise=9, dropped=4, degradation=5, perpendicular=4, outlier=9 laps) and rejected by Recorded (0). The direction-decision test asserts both are internally deterministic and produce distinct outcomes.
- **Task 1 track carries no intermediate sectors.** Adding oval-crossing sector boundaries is geometrically risky; `completedSectorResults` is still captured + asserted (empty, equal) to satisfy the artifact contract, and Task 2's N-run `CompleteSectorReplayTest` assertion covers non-empty sector-result determinism with proper sector geometry.
- **DraftRecovery equivalence uses `ReplayFixtures.multiLapLoop`, not the simulated provider**, because `DraftRecoveryTest`'s saved Track uses the DEMO start/finish; the big-oval `CLEAN_10_LOOP` produces 0 laps on that geometry. The equivalence (resume == fresh) held either way; the fixture swap makes the check meaningful (≥1 lap).
