## Per-Core-Path Audit (D-41 scope)

### 1. Architecture boundaries (ARCH-01 / D-45) — PASS
- **Clean-room grep** over `shared/src/commonMain/.../{lap,ghost,session,track,storage,export}/*.kt` for `import androidx`, `import platform`, `import android.`, `import java.`: **zero hits** in algorithm files. The only matches are Okio in three storage/export **I/O** files (see F-02), which is the correct multiplatform persistence seam, not a platform-UI dependency.
- **Engine purity grep** over `lap/` for `Random`, `currentTimeMillis`, `System.`, `Clock.`, `nanoTime`, `kotlin.random`: **zero hits.** `LapEngine.onSample()` is pure with respect to its sample input; all wall-clock/ID generation is isolated behind injectable `now: () -> Long` in controllers. This is the foundation the determinism gate (D-25–D-28) rests on.
- **No network/analytics in `commonMain`** (grep for ktor/okhttp/retrofit/firebase/java.net/analytics: zero hits). Local-first confirmed.

### 2. Lap engine, sector/crossing logic (LAP-01..07) — PASS
- `LapEngine`, `CrossingDetector`, `TimingLines`, `CourseGeometryBuilder` carry the crossing/interpolation/direction-gate/cooldown/min-lap/accuracy/speed filters. Determinism over the full pipeline (laps + `completedSectorResults` + ordered `LiveDeltaSnapshot` sequence + `CourseCompatibilityKey`/direction) is pinned by `FullPipelineDeterminismTest` and the widened legacy replay/sector/direction tests (Plan 02). No finding.

### 3. Replay (GPS-05 / D-25–D-28) — PASS
- `SessionReplayDecoder` (Plan 01) decodes exported `TimingSessionPayloadV1` JSON → `LocationSample` list → `ReplayRunner`, reusing the same `FileSessionStore.canonicalJson` and the existing `*.toModel()` mappers (no second engine — AGENTS.md clean-room). It is exercised N-times in the determinism suite across export→decode→replay. No finding.

### 4. GPS Ready / quality / provider state (GPS-01..05, D-13/D-14) — PASS
- `ReadyGate.aggregateReady(...)` (Plan 03) is a pure decision over all seven D-14 inputs + wrong-course preflight, returning a sealed `ReadyState.Ready`/`NotReady(reasons)`. Motion/speed/heading are correctly excluded (D-15). Conservative thresholds (25 m / 15 000 ms / 1.0 Hz) are injectable and validated with `init { require(...) }`.
- `WrongCoursePreflight` rejects non-finite/out-of-range coordinates and malformed geometry as **data** (`Unavailable(reason)`), never an exception (`WrongCoursePreflight.kt:88–131`). No finding.

### 5. Raw-recording seam (D-16/D-17) — PASS
- `RawRecordingController` (Plan 03) streams samples + `GpsQualitySummary` to a `RawRecordingPayload` flagged `isDiagnosticRaw = true` with `lapCount = 0`; it constructs no `TimingSessionRecorder`/`LapEngine`/`LiveDeltaEngine`, so its output is structurally excluded from valid-session counting. No finding.

### 6. Session lifecycle / draft / recovery (SESS-01..05) — PASS
- `SessionController` gates production formal timing on `aggregateReady` (opt-in `requireReady=true` from the Drive UI; legacy call sites preserved). The wrong-course override path (D-18) runs before the Ready gate and intentionally bypasses it to consume captured evidence. `DraftRecoveryTest` now asserts resume == fresh-replay equivalence (Plan 02). No finding.

### 7. Storage / schema migration — PASS
- `LocalSessionStore`/`FileSessionStore`/`InMemorySessionStore`/`SchemaVersions`/`SchemaMigrations` use deterministic `LinkedHashMap`/`LinkedHashSet` iteration. `LoadResult.Corrupt` keeps malformed JSON as data, not a crash. Field evidence relies on Save (not Discard) promoting the draft — exercised by recovery tests. No finding.

### 8. Export (SESS-04/05, V10/V12, T-5.1-03) — PASS
- `ExportFileNames.sanitizeNameToken` strips path separators (`/`,`\`), traversal (`..`→`__`), control chars (`–`,``), and XML specials; applies a strict `[a-zA-Z0-9._-]` whitelist; trims leading/trailing dots/spaces/dashes/underscores; truncates to 50; falls back to `"unnamed"`. Tested against an adversarial name in `JsonExportTest.kt:271–287`.
- `LocalExportWriter` writes only under the app-private `exports/` directory, owns filename placement (no caller-supplied paths), and stages atomically via `.tmp` + `atomicMove`. Path-traversal threat **mitigated**. No finding.

### 9. Track profile / revision / direction / wrong-course (Phase 5) — PASS
- `TrackProfileController`, `CourseGeometryBuilder.directionalCourse`, `GhostCompatibility`, `WrongCoursePreflight` own direction/revision/wrong-course as config-over-geometry. `CourseDirectionReplayTest` (widened, Plan 02) asserts Recorded vs Reverse produce deterministic, distinct outcomes. No finding.

### 10. Ghost reference / delta confidence (GHOST-01..04) — PASS
- `LiveDeltaEngine`/`CourseProgressMatcher` own match/unmatch transitions and suppression to `--`; confidence suppression lives in the domain, not the UI. The ordered transition sequence is determinism-pinned (Plan 02). No finding.

### 11. Live timing UI state (PLAT-03..05, D-31/D-32) — PASS
- `DriveScreen`/`AppShell` read `timingRunSnapshot()`/`liveDelta()` and surface Ready/not-Ready + raw-recording indicators; they own no algorithm. UI tokenization (Plan 05) routed colors/type/spacing through `MaterialTheme` + `LocalSpacing` with no information-hierarchy change. No finding.
- **Orientation (Pitfall 8 / project memory):** `AndroidManifest.xml:17` locks `screenOrientation="portrait"`; `MainActivity.kt:42–51` applies only fixed `SCREEN_ORIENTATION_PORTRAIT`/`LANDSCAPE` from the user toggle and explicitly never `SENSOR_*`/`USER_*`. Manual orientation confirmed. No finding.

### 12. Licensing / privacy in core paths (ARCH-03/ARCH-04, T-5.1-04/05) — see THIRD-PARTY-LICENSES.md
- Grep for `doves`/`gpl` in `shared/src`: **zero hits** — clean-room intact (ARCH-04). No network/analytics in `commonMain` — location traces are local-only (ARCH-03 privacy). The license/privacy **documentation** (ARCH-03/ARCH-04) was the genuine open audit item and is delivered by this plan in `docs/THIRD-PARTY-LICENSES.md`.
