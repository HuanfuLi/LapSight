<objective>
Build the core end-to-end connection slice: an Android-only `GlassesBridge` that runs
the DAT session lifecycle (register-gated createSession → session STARTED →
addDisplay → DisplayState STARTED), then drives a single uniform ~2 Hz poll-and-push
loop reading the hoisted `SessionController` (D-05/D-06), and silently reconnects on
disconnect (D-11) while phone timing is unaffected. Proven with MockDeviceKit — no
hardware required.

The connection state and device list that the phone UI (07-05) consumes are defined
as PLATFORM-FREE shared types in `shared/commonMain` (`GlassesConnectionState` sealed
interface + `GlassesDeviceSummary` DTO); the `GlassesBridge` in `androidApp` maps DAT
session/metadata into them. This keeps the KMP dependency direction correct
(androidApp → shared) so 07-05's shared-UI `collectAsState` binding compiles.

Purpose: This is the first real end-to-end: live phone timing state reaches the
(mock) glasses passively with zero glasses interaction (MR-03), through the same
SessionController seam MR-01 requires. The render this beat is a minimal single-line
placeholder — the full 3-page renderer is 07-04's slice (MVP: thin end-to-end first,
then replace the stub).
Output: shared `GlassesConnectionState` + `GlassesDeviceSummary` (platform-free);
`GlassesBridge` (androidApp) + MockDeviceKit lifecycle/reconnect tests; bridge
constructed and torn down in MainActivity.
</objective>

  Wearables.initialize(context) once (Application/MainActivity onCreate).
  Wearables.createSession(SpecificDeviceSelector(deviceId)).fold(onSuccess=session -> {

     scope.launch { session.state.collect { if (STARTED && display==null) attachDisplay(session) } }
