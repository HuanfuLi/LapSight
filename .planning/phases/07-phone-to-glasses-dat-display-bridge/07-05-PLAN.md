<objective>
Add the phone-side connection UX so the user can actually use the bridge: a Settings
"Glasses" area for one-time registration / device selection / firmware+app-update
prompts (D-10/D-12), and a Drive-screen "Cast to glasses" toggle + connection status
chip + phone-side page selector (D-07/D-10/D-11). The phone-side page selector is the
GUARANTEED, MR-03-safe primary control (captouch stays experimental in 07-06). Ends
with the real-hardware HUD legibility/passivity gate.

Purpose: Completes the user-facing loop — register once, cast per session, switch
pages passively from the phone, and see silent-reconnect status without a mid-drive
block. This is where a real user first sees the HUD end-to-end.
Output: Settings Glasses card + Drive cast/status/page controls wired to the bridge;
real-glasses human-verify.
</objective>
