# Objective
Fix UAT Gaps 1 and 2: Ensure V2 `TrackProfile` lifecycle operations (Duplicate, Rename, Archive) synchronize with the V1 `ReviewIndex` so the Review UI accurately reflects duplicated and renamed tracks without breaking backward compatibility. Enforce maximum name length limits in the controller to prevent UI rendering issues with extremely long names. Update the Review UI's trace section to load geometry from V2 profiles so traces render correctly for duplicated tracks.

# Context
Gap 1 identifies that `ReviewScreen` uses `sessionStore.readIndex()` (V1 tracks) while `TrackProfileController` lifecycle operations only update the V2 `profiles-index.json`. Thus, renamed and duplicated V2 profiles do not appear or update in the Review list. Additionally, `ReviewScreen.TrackTraceSection` attempts to load V1 `TrackPayloadV1` files directly (`tracks/<id>.json`), which do not exist for duplicated V2 profiles, causing trace rendering to fail.
Gap 2 identifies that `TrackProfileController` lacks a length constraint on profile names.

# Tasks

## 1. Synchronize `ReviewIndex` on Profile Save (Gap 1)
- **`<read_first>`**: `shared/src/commonMain/kotlin/com/huanfuli/lapsight/shared/storage/FileSessionStore.kt`, `shared/src/commonMain/kotlin/com/huanfuli/lapsight/shared/storage/InMemorySessionStore.kt`
- **`<action>`**: 
  - In `FileSessionStore.saveProfile()`, after updating `profilesIndexPath`, call `readIndex()` and `upsertRows()` to update the V1 `ReviewIndex`. Add a `ReviewIndexRow` constructed from the profile metadata (`id = profile.profileId`, `type = ReviewEntryType.Track`, `name = profile.name`, `createdAtEpochMillis = profile.createdAtEpochMillis`, `source = profile.source`, `payloadPath = "profiles/${profile.profileId}.json"`). Write the updated index to `indexPath`.
  - Repeat the equivalent logic in `InMemorySessionStore.saveProfile()` to maintain test consistency.
- **`<acceptance_criteria>`**: Saving a V2 Profile adds or updates a `ReviewIndexRow` representing it, ensuring duplicated tracks and updated names immediately appear in `sessionStore.readIndex()`.

## 2. Enforce Name Length Limit (Gap 2)
- **`<read_first>`**: `shared/src/commonMain/kotlin/com/huanfuli/lapsight/shared/track/TrackProfileController.kt`
- **`<action>`**: 
  - Update `isSafeProfileName(name: String)` to enforce a maximum length limit of 50 characters: add `trimmed.length in 1..50` to the existing conditional checks.
- **`<acceptance_criteria>`**: `isSafeProfileName()` strictly rejects names exceeding 50 characters, preventing UI rendering overflow.

## 3. Support V2 Profiles in Trace Section (Gap 1)
- **`<read_first>`**: `shared/src/commonMain/kotlin/com/huanfuli/lapsight/shared/ui/ReviewScreen.kt`
- **`<action>`**: 
  - In `TrackTraceSection`, add `val profileResult = remember(rowId) { sessionStore.loadProfile(rowId) }` and extract the V2 profile: `val profile = (profileResult as? LoadResult.Loaded<TrackProfile>)?.value`.
  - Update the resolution of trace inputs to gracefully fallback to the V2 profile if the V1 track fails to load:
    - `val markingId = track?.sourceMarkingSessionId ?: profile?.latestRevision?.sourceMarkingSessionId`
    - `val referenceLine = track?.referenceLine ?: profile?.latestRevision?.referenceLine`
    - `val startFinish = track?.startFinish ?: profile?.latestRevision?.courseSetup?.startFinish`
    - `val sectors = track?.sectors ?: profile?.latestRevision?.courseSetup?.boundaries?.map { SectorLineDto(id = it.id, name = "Sector ${it.order}", order = it.order, pointA = it.pointA, pointB = it.pointB) } ?: emptyList()`
    - Pass this `sectors` fallback into `buildTrackTraceLayers(..., sectors = sectors)`.
  - Update the "Trace data unavailable" check from `if (samples.isEmpty())` to `if (samples.isEmpty() && referenceLine?.points.isNullOrEmpty())` to account for profiles with reference lines but no explicitly loaded samples.
- **`<acceptance_criteria>`**: When expanding a duplicated V2 track (which lacks a V1 payload), `TrackTraceSection` extracts geometry from the V2 profile and successfully renders its trace.
