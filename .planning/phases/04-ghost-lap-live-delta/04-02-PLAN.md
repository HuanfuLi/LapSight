## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Saved session payloads -> reference selection | Historical data may be simulated, corrupt, or for another Track. |
| Active session -> global reference | User may discard a session after producing a faster lap. |
| Storage -> timing recorder | Missing/corrupt reference must not block timing. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-04 | Integrity | reference selection | mitigate | Filter by Track ID and source boundary; tests assert real/sim separation. |
| T-04-05 | Tampering/DoS | ghost reference JSON | mitigate | Versioned schema, typed load results, and safe unavailable state on corrupt payload. |
| T-04-06 | Repudiation/Integrity | Save/Discard lifecycle | mitigate | Persist global reference only after explicit Save; discard leaves stored reference unchanged. |
