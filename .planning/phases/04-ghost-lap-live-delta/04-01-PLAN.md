<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GPS samples -> progress curve | Raw samples can be noisy, sparse, non-monotonic, or inaccurate. |
| Reference curve -> live delta | A stored reference may be missing, corrupt, too short, or incompatible with current progress. |
| Algorithm -> UI | Delta output must not imply precision when confidence is low. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Integrity/Safety | progress curve | mitigate | Validate finite coordinates, monotonic timestamps, minimum points, and positive total distance; return unavailable/failure for bad inputs. |
| T-04-02 | Tampering/DoS | reference curve interpolation | mitigate | Binary search/clamping must handle edge cases without crashing or producing stale values. |
| T-04-03 | Safety | delta semantics | mitigate | Suppress delta under poor confidence and keep `--` for unavailable states; tests assert no stale delta. |
</threat_model>
