## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `GpsFixtureLibrary` fixture scenarios | mitigate | Keep fixtures deterministic, generated from fixed seeds, and covered by tests that assert monotonic timestamps, finite coordinates, and `LocationSource.Simulated`. |
| T-03-02 | Information Disclosure | committed fixture JSON files | mitigate | Use only synthetic GPS coordinates and scenario metadata; do not add real user traces. |
| T-03-03 | Spoofing | Demo source labeling | mitigate | Persist and render `LocationSource.Simulated` and visible `DEMO - simulated GPS` labels so simulated data cannot appear as live GPS. |
| T-03-04 | Integrity | future ghost candidate inputs | mitigate | Mark every generated sample as simulated; later plans must filter simulated sessions out of real ghost-candidate calculations per D-43. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | This plan adds no package-manager dependencies. Dependency risk is handled by the blocking package checkpoint in Plan 03-02. |
</threat_model>
