<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI -> track marking controller | User starts/stops marking and chooses Save/Re-record/Discard. |
| Provider samples -> reference extraction | Untrusted sample quality, outliers, and malformed values feed a pure extraction algorithm. |
| Shared storage -> local filesystem | Canonical GPS payloads are written and loaded from app-private storage. |
| Local JSON -> domain models | Saved payloads can be malformed or corrupt on load. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information Disclosure | `FileSessionStore` canonical payloads | mitigate | Store payloads under app-private roots supplied by `StoragePaths`; export copies are handled only in Plan 03-06. |
| T-03-07 | Tampering/DoS | `FileSessionStore` JSON decode | mitigate | Validate `schemaVersion`, required IDs, finite coordinates, monotonic elapsed times, and source metadata; return corrupt/quarantined load results instead of crashing. |
| T-03-08 | Tampering | atomic file writes | mitigate | Write temp payloads, atomically move payloads, then update `ReviewIndex`; tests inject write failures. |
| T-03-09 | Integrity | `ReferenceLineExtractor` | mitigate | Use fixture-backed tests for outlier/noise/dropped samples and preserve raw marking samples for recomputation. |
| T-03-10 | Integrity | Demo review rows | mitigate | Persist and render `LocationSource.Simulated`/`DEMO` badges on saved tracks and marking payloads. |
| T-03-SC | Tampering | Maven/Gradle dependencies | mitigate | Add only Plan 03-02 approved coordinates; no package substitutions during implementation. |
</threat_model>
