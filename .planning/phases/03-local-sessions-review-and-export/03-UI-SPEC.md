# Phase 3 — UI Design Contract

> Visual and interaction contract for Local Sessions, Review, and Export.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Platform: Kotlin Multiplatform + Compose Multiplatform (Android/iOS). This is a
> mounted racing-dash / motorsport telemetry app. shadcn does not apply (no web
> stack); the contract extends the existing dark dash theme already shipped in
> `shared/src/commonMain/.../App.kt`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Compose Multiplatform (not shadcn; no React/Next/Vite) |
| Preset | not applicable |
| Component library | Compose Material3 `1.11.0-alpha07` (existing). Use `NavigationBar` / `NavigationBarItem` for the 3-tab shell, `Card`, `Button`, `ModalBottomSheet`, `AlertDialog`. |

## Spacing Scale

Declared values (4-point base; all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline unit padding |
| sm | 8px | Compact element spacing, chip gaps |
| md | 12px | Card inner padding (compact), metric grid gaps |
| lg | 16px | Default element spacing, card padding, list-row vertical rhythm |
| xl | 24px | Section padding, sheet/dialog padding |
| 2xl | 32px | Major section breaks |
| 3xl | 48px | Page-level spacing, fullscreen Drive margins |

**Exceptions:**
- **Minimum interactive target: 48px** for all tappable controls (buttons, nav items, list rows, Save/Discard). Mounted-phone, gloved, in-motion use forbids small controls (SAFE-02, D-29).
- The existing Phase 2 dash uses some odd transitional values (6/10/14/18/20dp). New Phase 3 screens MUST snap to the scale above; do not introduce new non-multiple-of-4 values. Migrate legacy values opportunistically.

## Typography

Mounted-dash, glanceable-while-moving hierarchy (SAFE-01: large type, high contrast). Four roles, two core weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 52px (40px in compact landscape) | Black 900* | 1.0 | Fullscreen Drive/Timing primary readout only (current lap, latest sector). Tabular figures required. |
| Heading | 28px | Bold 700 | 1.2 | Screen titles, secondary metric values (last/best/laps), Review detail headers |
| Body | 16px | Regular 400 | 1.5 | List rows, session summaries, safety copy, dialog body, settings |
| Label | 13px | Bold 700 | 1.3 | Metric captions, tab labels, chips, source/Demo badges, uppercase section keys |

**Weights declared:** Regular 400 + Bold 700.

\*Black 900 is the single documented weight exception, reserved **exclusively** for the Display role on the fullscreen Drive/Timing surface. Rationale: SAFE-01 requires maximum-glanceability timing numerals at speed; this matches the shipped Phase 2 dash. It must not appear on Review, Settings, or any body/label text.

## Color

Dark-first racing theme (extends existing `darkColorScheme` in `App.kt`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#05070A` | App background, fullscreen Drive surface, screen scaffolds |
| Secondary (30%) | `#101722` | Cards, metric panels, bottom NavigationBar, bottom sheets, list rows |
| Accent (10%) | `#62E3FF` (cyan) | See reserved list below |
| Destructive | `#FF6B6B` (red) | Discard / delete actions and their confirmation only |

**Accent (`#62E3FF`) reserved for — explicit list (never "all interactive elements"):**
1. Active timing state indicator on the Drive surface
2. Primary metric highlight (current lap value, latest sector split)
3. Selected bottom-nav tab (icon + label)
4. The single primary CTA per screen (Start Timing / Save Track / Save Session / Mark New Track)
5. "Simulated" GPS source status text

Secondary buttons, Resume/Re-record, export buttons, and inactive nav tabs use neutral surface/outline styling — NOT accent.

**Status / semantic sub-palette** (required for GPS quality summary, SESS-02, and source labeling; already present in `App.kt`):

| Token | Value | Meaning |
|-------|-------|---------|
| Good / Live | `#8CFF9B` (green) | Good GPS quality, live fix |
| Caution / Acquiring | `#FFD166` (amber) | Acquiring, degraded-warn, **Demo/Simulated badge** |
| Degraded | `#FFB84D` (orange) | Degraded fix / low sample rate |
| Bad / Unavailable | `#FF6B6B` (red) | No fix / unusable quality (shares Destructive hue) |
| Muted text | `#9AA8B8` | Secondary text, units |
| Caption text | `#7E8DA0` | Labels, captions, helper text |
| Body text | `#CED7E2` | Primary body copy on dark surfaces |

**Demo/Simulated labeling (D-42, D-43):** Every simulated track, session, Review row, and export-origin badge shows a `DEMO` pill — Label type, amber `#FFD166` text on `#101722`, 1px amber outline. Demo data must be visually distinct from real data in every list and detail screen.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Drive (no track): **"Mark New Track"** · Drive (track ready): **"Start Timing"** · Track Review: **"Save Track"** · Stop summary: **"Save Session"** |
| Empty state heading | Review list: **"No saved sessions yet"** |
| Empty state body | **"Mark a track, then start a timing session. Saved tracks and sessions show up here."** |
| Error state | Low GPS quality: **"Low GPS quality — lap times may be unreliable. Check sky view and accuracy before trusting splits."** |
| Destructive confirmation | Discard session: **"Discard this session? Recorded laps will be lost. This can't be undone."** — buttons `Discard` (destructive) / `Keep` |

**Extended Phase 3 copy (load-bearing for SESS-01..05, D-05, D-14, D-15, D-31, D-44):**

| Surface / State | Copy |
|-----------------|------|
| Demo feed toggle (off → on) | **"Start Demo Feed"** / when on: **"Stop Demo Feed"** (small, clear control — D-44) |
| Drive while demo running | Banner: **"DEMO — simulated GPS"** (amber pill) |
| Start blocked, no track | **"Mark a track first. Timing needs a saved start/finish line."** |
| Stop summary sheet title | **"Session ended"** — primary `Save Session`, secondary `Discard` |
| Save success toast | **"Session saved"** (and **"New track best"** when applicable — D-32) |
| Draft recovery on launch (D-15) | Title: **"Unfinished session found"** · Body: **"You have a session that wasn't saved."** · Actions: `Resume` / `Save` / `Discard` |
| Track Review — not save-ready (D-31) | **"Couldn't build a clean track. Re-record 5–10 continuous loops and avoid long stops."** |
| Track Review — ready | **"Track ready. Set start/finish, then save."** |
| Sectors empty (existing) | **"No sectors on this course."** |
| Export menu | **"Export JSON"** (canonical/debug) · **"Export GPX"** (for other GPS tools) |
| Export success | **"Exported {filename}"** |
| Export failure | **"Export failed. Check device storage and try again."** |
| Safety note (persistent, SAFE-03 carryover) | **"Closed-course use only. Phone GPS accuracy varies — this is not pro-grade timing."** |

**Destructive actions in this phase + confirmation approach (all use Material3 `AlertDialog`, destructive button in `#FF6B6B`):**

| Action | Confirmation copy | Buttons |
|--------|-------------------|---------|
| Discard timing session (Stop summary, D-14) | "Discard this session? Recorded laps will be lost. This can't be undone." | `Discard` / `Keep` |
| Discard unfinished draft (launch prompt, D-15/D-16) | "Discard unfinished session? Recorded data will be lost." | `Discard` / `Cancel` |
| Re-record track (Track Review, D-31) | "Re-record track? The current marking trace and reference line will be replaced." | `Re-record` / `Cancel` |
| Discard track (Track Review) | "Discard this track? You'll lose the marking trace and reference line." | `Discard` / `Cancel` |

Copy rules: verb-first buttons, no "Are you sure", state the consequence and reversibility. Destructive verb is never the default/accent-styled button.

## Trace Rendering Contract (Phase-specific, SESS-03 / D-33–D-36)

Offline vector trace via Compose `Canvas` (no map tiles). Stroke + color contract:

| Element | Color | Stroke |
|---------|-------|--------|
| Track reference line (baseline) | `#62E3FF` accent | 3px |
| Full marking trace / session trace (context) | `#9AA8B8` muted | 2px |
| Outlier / rejected sections (Track Review) | `#FFB84D` orange, 50% alpha | 2px dashed |
| Start/finish line | `#8CFF9B` green | 3px |
| Sector lines | `#FFD166` amber | 2px |
| Selected / best lap highlight (hook) | `#62E3FF` accent | 4px |

Canonical lat/lon stays in saved data; Canvas receives projected screen points only (D-34).
