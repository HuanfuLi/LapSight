## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Review UI -> export services | User-selected saved entity IDs trigger export generation. |
| Saved canonical payloads -> export artifacts | Sensitive GPS traces are transformed into files intended for external use. |
| User-controlled track/session names -> filenames/XML | Names enter filenames and GPX metadata. |
| Export artifacts -> app-private export directory | Export files are written separately from canonical payload files. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-19 | Information Disclosure | JSON/GPX export actions | mitigate | Exports require explicit Review-detail button taps and never run automatically or from batch list actions. |
| T-03-20 | Tampering | `ExportFileNames` | mitigate | Generate filenames internally, strip path traversal/separators/control characters, and write only under app-private `exports/`. |

| T-03-23 | Integrity | Demo export metadata | mitigate | Preserve source metadata and visible Demo/Simulated labels in exported JSON/GPX metadata where applicable. |
