# Sketch 002: Track Center List

## Design Question

How should a fullscreen Track Center prioritize recent and nearby tracks while
keeping global database search and custom-track recording available but quiet?

## How to View

```bash
open .planning/sketches/002-track-center-list/index.html
```

## Variants

- **A: Sectioned Utility** — recent and nearby are explicit sections; database search is a fixed bottom action.
- **B: Ranked Library** — one compact relevance-ranked list with reason labels and a top search launcher.
- **C: Recent First** — recent tracks use stronger visual cards; nearby remains a dense secondary list.

## Winner

**A: Sectioned Utility.** Recent and nearby remain explainable, database search
is available in the bottom thumb zone without dominating normal use, and custom
track creation stays in the overflow menu.

## What to Look For

- Which hierarchy makes the common “reuse my last track” flow fastest.
- Whether nearby tracks are discoverable without feeling like a social feed.
- Whether search is available without dominating normal use.
- Whether custom-track creation is appropriately secondary.
