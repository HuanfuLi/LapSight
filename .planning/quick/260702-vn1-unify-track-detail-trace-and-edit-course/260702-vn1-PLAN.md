# Quick Task 260702-vn1: Unify Track Trace and Edit Course Map

## Goal

Fix the Track detail duplication introduced by the field-test hardening pass:

- The original `Trace` map must use the improved circuit styling.
- Entering `Edit course` must not add a second identical map.
- The single Track map should switch between browse mode and edit mode in place.

## Scope

In scope:

- Extract a reusable beautified course-map canvas from `TrackEditorScreen`.
- Use that shared canvas for Track detail `Trace`.
- Move edit controls so they operate on the same map instead of rendering another `TrackEditorScreen`.
- Preserve existing start/finish and sector drag behavior from quick task `260702-uju`.
- Verify with host tests and Android debug build, then deploy to the connected device with ADB/Gradle.

Out of scope:

- Timing-session replay chart/map redesign.
- Raw capture map redesign unless needed for compile reuse.
- Lap engine or storage model changes.

## Acceptance Criteria

- A Track detail row shows one course map, not two.
- The visible Track detail map uses the improved thick circuit style.
- Pressing `Edit course` turns that same map into editable mode with handles.
- Save/Cancel exits edit mode without leaving a duplicate map behind.
- Revision save still refreshes the Track detail immediately.
- `.\gradlew.bat :shared:testAndroidHostTest` passes.
- `.\gradlew.bat :androidApp:assembleDebug` passes.
- The resulting debug APK is installed and launched on the connected Android device.
